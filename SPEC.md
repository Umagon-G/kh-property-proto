## 2. 技術スタック
- 単一 HTML 自己完結（ビルド不要・依存は CDN のみ）。
- 地図: **Leaflet** + OpenStreetMap タイル（Google Maps 化は API キー差替で可能・§7 backlog）。
- クラスタ: **leaflet.markercluster**（`chunkedLoading`・行政区域単位の自前 icon）。
- エリア囲い判定: **turf.js**（`booleanPointInPolygon`）。
- 描画: `preferCanvas:true`（1,700 マーカーの描画負荷対策）。
- 乱数: `mulberry32` 固定シード **20260618**（誰が開いても同じ物件が再現される＝デモの一貫性）。

## 4. 機能仕様
### 4-1. エリア囲い検索（中核UX）
- 「✍指でなぞって囲む」ON → 地図ドラッグで自由曲線 → 離すと閉ポリゴン化 → turf で内側の物件だけ抽出。
- pointer イベント（タッチ/マウス両対応）。`map.dragging` を描画中だけ無効化。
- 「デモエリア」ボタン = BKK1 付近に自動で円を描く（録画用ワンタップ）。

### 4-2. 絞り込み（**全軸が複数選択**）
| フィルタ | 方式 |
|---|---|
| 取引 | 複数選択（賃貸 / 売買）。Set 方式 |
| 用途 | 複数選択（住居 / 事業用 / 土地） |
| 物件タイプ | 複数選択（10種） |
| こだわり条件 | 複数選択（検証済 / 税インボイス可 / 専用メーター / 固定家賃） |
| 価格上限 | スライダー（賃料↔総額を連続でまたぐ） |
| 面積下限 | スライダー |
- **「すべて」= その軸の Set をクリア（＝全表示）**。個別タップで ON/OFF トグル。複数 ON 可。
- **ヘッダーの取引/用途トグル と 絞り込みパネルのチップ は同じ Set を共有**し相互同期（`applyMultiClass`）。
- 選択中は **✓マーク** 表示＋ヘッダーに「複数選択OK」バッジ（複数選べると一目で分かる affordance）。
- スライダーはドラッグ中は件数のみ更新、離した時に地図反映（再描画負荷の抑制）。

### 4-3. 行政区域クラスタ
- markercluster の icon を自前生成。**色は件数で変えず統一ネイビー** `#0B2138`。
- クラスタに **区域名＋件数** を表示。サイズのみ件数で可変。
- 個別ピンの色 = **用途**（住居=青 `#1d4ed8` / 事業用=ティール `#0E7C86` / 土地=緑 `#0E9F6E`）。
- **売買は太いアンバー枠**（`#F4A300`・weight 3）で賃貸と区別。凡例つき。

### 4-4. リスト / ソート
- 左カラムは**スクロール可**（flex レイアウト・上位 `LIST_CAP`=80 件表示＋「他N件は地図で確認」）。
- ソート: おすすめ順(rec) / 価格安い順(pa) / 価格高い順(pd) / 面積広い順(ad)。
- カード ↔ 地図ピン 相互ハイライト（クリックで該当へスクロール／地図移動）。

## 5. カラー / デザイン原則
- ベース: ネイビー `--navy:#0B2138` / アンバー `--amber:#F4A300` / インク `#1a2b3c`。
- **クラスタは統一色**（カラフルにしない＝白石指摘）。色は「用途の区別」と「売買=太枠」にのみ使う。
- マーケ的装飾を避け実用主義（情報密度優先）。
